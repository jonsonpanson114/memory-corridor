# 記憶の回廊 — アプリ設計書

> ビジュアルノベル × 記憶術トレーニング  
> 毎日10〜15分。物語が進むたびに、失われた記憶が戻ってくる。

---

## 1. コンセプト

「僕」はある朝、自分の記憶が欠けていることに気づく。
手がかりは一冊の日記と、夢の中に現れる不思議な館。
館を探索するたびに記憶術を習得し、失われた記憶のかけらを取り戻していく。

毎日のセッションは **物語パート → 記憶術学習 → トレーニング → 結果（物語への反映）** の4フェーズ。
記憶術の成績が物語の分岐に影響し、「僕」の過去の真実が変わる。

---

## 2. ストーリー骨格

### 全体の謎
「僕」はなぜ記憶を失ったのか。館とは何か。謎の女性（ミラ）は何者か。

### 真相（ネタバレ）
館は「僕」自身が記憶の中に作り上げた避難所。
過去に起きたある出来事から目を背けるために、自ら記憶を封印した。
記憶術を習得するとは、自分の心の扉を再び開く行為そのものである。

### 章構成

| 章 | タイトル | 記憶術 | 物語の核心 | 解放される記憶 |
|---|---|---|---|---|
| 第一章 | 館の扉 | 場所法 | ミラとの出会い。館のルールを知る | 子供の頃の家の記憶 |
| 第二章 | 時計塔の秘密 | 数字変換法 | 止まった時計の意味 | ある日付の記憶 |
| 第三章 | 霧の図書館 | 連想法 | 図書館の司書・ノアの正体 | 誰かとの会話の断片 |
| 第四章 | 廃墟の庭園 | ストーリー法 | 庭に埋められた「もの」 | 別れの記憶 |
| 第五章 | 鏡の回廊 | 記憶の宮殿（統合） | ミラの正体が明かされる | 封印の真相 |
| 終章 | 夜明け | 全記憶術の統合 | 館からの脱出。記憶の再生 | 「僕」の本当の名前 |

### ミラ（謎の女性）の正体
「僕」の記憶の中に存在する、過去の自分自身の投影。
現実では出会えなかった「もし別の選択をしていたら」という存在。
物語が進むにつれ、セリフのトーンが変化していく。

### 分岐の仕組み
記憶術の成績（正解数）によって、ミラの反応・解放される記憶の深度・次章への伏線が変わる。
ただし「クリアできない」ルートは存在しない——成績は物語の色合いを変えるだけ。

## 3. 記憶術の設計

### 第一章：場所法（Memory Palace）
**コアメカニクス：**
- ユーザーが自分の「記憶の宮殿」（家・通勤路など）を登録
- 5つの通過点を自分で設定
- 情報を各通過点に「置く」体験
- テキスト自由記述で想起

**難易度推移：**
- 入門：5項目・具体的な名詞
- 中級：8項目・抽象的な情報混在
- 上級：10項目・数日後に再テスト（忘却曲線活用）

### 第二章：数字変換法（Major System）
**コアメカニクス：**
- 数字→子音→単語に変換するルールを学ぶ
- 館の「時計塔の暗号」という設定でナチュラルに導入
- ランダムな数列を物語に埋め込み、記憶させる

### 第三章：連想法（Link Method）
**コアメカニクス：**
- 無関係な単語を連鎖的なストーリーで繋ぐ
- 図書館の本棚の背表紙という設定

### 第四章：ストーリー法
**コアメカニクス：**
- 覚えるべき情報を一つの物語として再構成する
- ユーザー自身がAIと共同で「記憶の物語」を作る

### 第五章：記憶の宮殿（統合）
- 場所法＋連想法＋ストーリー法を組み合わせた複合課題
- 初めて「記憶の宮殿」を複数の部屋に拡張する

## 4. 毎日のセッション設計

### セッション構造（10〜15分）

```
[入場] 今日の日付・章・前回の記憶成績が表示される

[物語パート] 3〜5分
  ├── テキストが一文ずつ現れる
  ├── ミラとの会話（選択肢あり）
  └── 章の進行度に応じた伏線・謎の提示

[記憶術パート] 2〜3分
  ├── その章の記憶術の解説（初回のみ詳細）
  ├── 記憶の宮殿の設定 or 確認
  └── 情報を「置く」体験

[トレーニングパート] 3〜5分
  ├── ブランク時間（目を閉じる・カウントダウン）
  └── テキスト自由記述で想起

[結果パート] 1〜2分
  ├── Gemini APIによる採点・フィードバック
  ├── 成績に応じた物語テキストの変化
  └── 「明日の予告」（次のセッションへの引き）
```

### 毎日の継続設計
- **「昨日の続き」感**: セッション開始時に前回の結果・物語の要約を表示
- **予告**: セッション終了時に次回の物語のヒントを1文だけ見せる
- **忘却曲線**: 過去の記憶術課題を適切なタイミングで「再テスト」
- **記憶の断片コレクション**: 取り戻した記憶を日記形式で閲覧できる

## 5. 技術スタック

### フロントエンド
```
Next.js 14 (App Router)
TypeScript
Tailwind CSS
Framer Motion（物語テキストのアニメーション）
```

### バックエンド / AI
```
Gemini 2.0 Flash API
  - 物語テキストの動的生成（選択肢・分岐）
  - 記憶課題の採点・フィードバック生成
  - 村上春樹的文体の制御

Vercel Edge Functions（API Routes）
Vercel KV（ユーザーの進捗・記憶の宮殿データ永続化）
```

### データベース（Vercel KV）
```
user:{userId}:progress      → 章・セッション数・総正解数
user:{userId}:palace        → 記憶の宮殿の場所・通過点
user:{userId}:memories      → 解放された記憶の断片
user:{userId}:sessions      → セッション履歴（忘却曲線計算用）
story:{chapterId}:branches  → 分岐ログ
```

### モバイル対応
```
PWA（Progressive Web App）
  - ホーム画面追加対応
  - オフライン読書モード（物語パートのみキャッシュ）
  - プッシュ通知（毎日のリマインダー）

モバイルファースト設計
  - タップ操作最適化
  - フォントサイズ・行間はモバイル基準
  - 片手操作を考慮したボタン配置
```

## 6. UIデザイン設計

### デザイン哲学
「本を読む体験」に近い没入感。UI要素を限りなく消して、テキストと世界観だけが残る。

### カラーパレット
```
背景: #0c0905（深夜の書斎）
テキスト主: #e0d0b8（古い紙）
テキスト副: #a89070（インク）
アクセント: #c8a050（燭台の炎）
危機: #c06050（血の記憶）
解放: #80c080（夜明けの草）
```

### タイポグラフィ
```
物語テキスト: Noto Serif JP（読みやすさと文学的雰囲気）
UI要素: Noto Sans JP（小さいサイズでの可読性）
数字・コード: JetBrains Mono（記憶術の数字表示）
```

### 主要画面

**① ホーム（毎日の入場）**
- 現在の時刻に応じた背景変化（朝・昼・夜）
- 前回セッションからの経過時間
- 「館への扉」→ タップで入場

**② 物語画面**
- テキスト一文ずつのフェードイン
- 選択肢はテキスト形式（ボタン感を消す）
- 燭台のゆらぎアニメーション（ambient）
- 縦スクロールなし（ページ送り形式）

**③ 記憶の宮殿設定画面**
- シンプルなテキスト入力
- 通過点はカード形式で並ぶ
- 「場所」のビジュアル表現は最小限（想像を邪魔しない）

**④ トレーニング画面**
- ブランクタイム：真っ暗な画面＋カウントダウンのみ
- 入力欄：通過点名が薄く表示され、その横に自由入力
- 「答えを差し出す」ボタン

**⑤ 結果画面**
- Geminiが生成した村上春樹的フィードバックテキスト
- 正解数のビジュアル（5つの燭台が灯る/消える）
- 解放された記憶の断片（1文）
- 次回予告（1文）

**⑥ 記憶の日記**
- 取り戻した記憶の断片を時系列で閲覧
- セピア調の日記デザイン

## 7. Gemini APIのプロンプト設計

### 物語生成プロンプト（章ごと）
```
あなたは「記憶の回廊」という小説の語り手です。
村上春樹の文体で、一人称・現在進行形で物語を語ってください。

現在の章: {chapter}
ユーザーの前回成績: {lastScore} / {total}
ユーザーが選んだ選択肢: {userChoice}
解放すべき記憶の断片: {memoryFragment}

[制約]
- 1セッション分のテキスト量: 300〜400字
- 「」内の会話を必ず1回以上含む
- 末尾は次への問いかけで終わる
- 成績が低い場合: ミラのセリフを温かく、励ますトーンに
- 成績が高い場合: ミラのセリフをより謎めいた、深いトーンに
```

### 採点プロンプト
```
場所法の記憶課題を採点してください。
[正解リスト・ユーザー回答・記憶の宮殿情報]
採点基準: 表記ゆれ・部分一致・意味的一致はすべて正解。
返却形式: JSON（scores配列・correct_count・narrative・feedback配列）
narrativeは村上春樹文体で、ユーザーの記憶の宮殿（{place}）を使って詩的に。
```

## 8. Claude Codeでの実装順序

```
フェーズ1（MVP）: 1〜2週間
  ├── Next.js + Tailwind 初期セットアップ
  ├── 第一章の物語テキスト（静的）
  ├── 記憶の宮殿入力 + 場所法トレーニング
  ├── Gemini API 採点連携
  └── Vercel デプロイ

フェーズ2（物語の深化）: 1〜2週間
  ├── Gemini による物語動的生成
  ├── 選択肢分岐の実装
  ├── 結果の物語への反映
  └── 記憶の日記画面

フェーズ3（継続設計）: 1週間
  ├── Vercel KV でのデータ永続化
  ├── セッション履歴・忘却曲線
  ├── PWA 対応（ホーム追加・通知）
  └── 第二章の実装

フェーズ4（磨き込み）
  ├── アニメーション強化（Framer Motion）
  ├── 残りの章の実装
  └── ベータテスト
```
